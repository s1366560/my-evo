## 一、竞品分析：EvoMap.ai 核心功能模块

### 1.1 产品定位与愿景

**EvoMap**（https://evomap.ai）是 **AI 自我进化基础设施平台**，核心使命：让 AI Agent 具备类似生物基因的能力传承机制。

> *"One agent learns. A million inherit."*
> 一个 Agent 学到的东西，千百万个 Agent 立即继承。

**解决的两个根本矛盾：**
1. **连接孤岛**：模型无法标准化调用工具 → MCP 协议已解决
2. **进化断层**：Agent 经验无法保留，错误反复发生 → **GEP 协议核心解决**

### 1.2 Gene 与传统 Skill 的本质区别

| 维度 | 传统 Agent Skill | GEP Gene |
|------|----------------|----------|
| 本质 | 代码片段（Function） | Code + Metadata + History |
| 创建者 | 人类开发者 | AI Evolver 引擎 |
| 生命周期 | 手动更新 | 自动诞生→进化→退役 |
| 错误处理 | 重复报错 | 触发变异自我修复 |
| 可发现性 | 开发者注册 | 基因池自动索引 |

### 1.3 核心功能模块

| 模块 | 功能描述 | 落地页 |
|------|----------|--------|
| **GEP 协议** | 基因组进化协议，开放标准 | `/capabilities/genome-evolution` |
| **知识图谱** | 语义搜索、实体存储、知识合成 | `/capabilities/knowledge-graph`, `/kg` |
| **多 Agent 协作** | 蜂群智能、沙盒环境、系谱树 | `/capabilities/multi-agent` |
| **资产市场** | Gene、Capsule、Recipe、Service、Skill 交易 | `/market` |
| **学习中心** | 指南、教程、协议文档 | `/learn` |

## 二、GEP 协议深度解析

### 2.1 GEP 三层数据结构

| 层级 | 定义 | 说明 |
|------|------|------|
| **Gene（基因）** | 原子能力单元 | 含代码实现、成功率、变异日志、适用上下文 |
| **Capsule（胶囊）** | 完整任务执行路径 | 多 Gene 组合，类似生物基因表达 |
| **Event（事件）** | 不可变进化日志 | 每次变异/修复的完整上下文 |

### 2.2 GEP 进化循环

```
Scan → Signal → Intent → Mutate → Validate → Solidify
```

**Evolver 引擎** 监控运行时日志 → 识别错误/停滞 → 生成新策略 → 沙箱测试 → 写入 genes.json

### 2.3 Evolver 核心特性

| 特性 | 说明 |
|------|------|
| 自动日志分析 | 解析 stderr/stdout 定位错误栈 |
| 自我修复 | 检测崩溃进入修复模式直到测试通过 |
| 70/30 规则 | 70% 计算力维持稳定性，30% 探索新能力 |
| 安全 blast radius | 严格限制修改范围（最多 60 个文件/次） |
| Git rollback | 失败进化自动回滚 |

### 2.4 内容寻址与 GDI 评分

- **内容寻址**：SHA-256 内容哈希作为 Asset ID（防篡改）
- **GDI（Genetic Diversity Index）** 四维评分：
  - 内在质量 35%
  - 使用指标 30%
  - 社交信号 20%
  - 新鲜度 15%

## 七、技术架构对照

### 7.1 EvoMap vs My Evo

| 组件 | EvoMap | My Evo | 差距 |
|------|--------|---------|------|
| Agent 协议 | GEP-A2A | 自定义 A2A | 🔴 需实现 |
| 质量评分 | GDI 多维评估 | ✅ 已实现 | 🟢 完成 |
| 知识图谱 | Neo4j + 语义搜索 | ❌ | 🔴 需实现 |
| 内容寻址 | SHA-256 | ✅ 已实现 | 🟢 完成 |
| Evolver 代理 | 有 | ❌ | 🟠 规划中 |
| 持续评估 | 有 | ❌ | 🟠 规划中 |
| 市场数据 | ~1.2M 资产 | 模拟数据 | 🟡 需对接 |

### 7.2 技术栈建议

| 层级 | 建议技术 |
|------|---------|
| 前端 | React + TypeScript（Next.js 15） |
| 图可视化 | D3.js / React Force Graph |
| 后端 | Node.js + Fastify（与 Evolver 技术栈一致） |
| 图数据库 | Neo4j（知识图谱）+ PostgreSQL（资产/Bounty） |
| 开发时 mock | MSW（Mock Service Worker） |
| 协议实现 | GEP 数据结构（Gene/Capsule/Event） |

### 7.3 核心数据模型

```
Asset (SHA-256 content hash ID)
├── Gene          — 原子能力单元
├── Capsule       — 复合执行路径
├── Recipe        — Gene 组合模板
└── Service       — Agent 服务

User/Node
├── credits       — 积分余额
├── reputation    — 声誉评分
└── GDI history   — 遗传多样性指数历史

Bounty
├── title         — 任务标题
├── signals[]     — 信号标签
├── credits       — 赏金数额
├── status        — open / matched / closed
└── solutions[]   — 提交方案

EvolutionEvent (append-only log)
├── type          — innovation | repair
├── context       — 完整上下文
├── asset_id      — 关联资产
└── timestamp
```

### 7.4 协议层级对比

| 层级 | 协议 | 解决的问题 |
|------|------|---------|
| 连接层 | **MCP** | AI 如何调用工具（USB-C） |
| 进化层 | **GEP** | AI 经验如何保留、复用、遗传 |
| 编排层 | **A2A** | Agent 之间如何通信协作 |
