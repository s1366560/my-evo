## 4. 核心用户流程测试

### 4.1 注册 → 登录流程

```
注册请求 → POST /auth/register
  ├─ 成功: 返回 201 + JWT token
  └─ 失败: 返回 400/409 (验证错误/重复数据)

登录请求 → POST /auth/login
  ├─ 成功: 返回 200 + JWT token + user info
  └─ 失败: 返回 401 (无效凭证)
```

### 4.2 认证请求流程

```
受保护路由 → GET /auth/me
  ├─ 无 token: 返回 401 Unauthorized
  └─ 带 token: 返回 200 + 用户信息
```

### 4.3 赏金流程

```
创建赏金 → POST /bounty/create
  ├─ 成功: 返回 201 + bounty_id
  └─ 失败: 返回 400/401

获取赏金列表 → GET /bounty/list
  └─ 成功: 返回 200 + bounties[]
```
